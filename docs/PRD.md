# AI PPT Creator - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
**AI PPT Creator** - 智能演示文稿生成器

### 1.2 产品定位
一款基于AI的在线PPT自动生成工具，用户只需输入主题、选择语言和设定演讲时长，AI即可自动收集资料、规划大纲并生成专业的演示文稿。

### 1.3 目标用户
- 需要快速制作演示文稿的职场人士
- 准备课堂演讲的学生
- 需要制作汇报材料的研究人员
- 任何需要快速生成PPT的用户

### 1.4 核心价值
- **省时高效**：将数小时的PPT制作时间缩短至几分钟
- **智能内容**：AI自动搜集相关资料，确保内容专业准确
- **零门槛**：无需设计能力，自动生成美观排版

## 2. 功能需求

### 2.1 核心功能流程

```
用户输入 → AI收集资料 → 规划大纲 → 生成PPT → 导出PPTX
```

### 2.2 功能模块详述

#### 2.2.1 输入模块

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 主题 | 文本输入 | ✓ | 演示文稿的核心主题，支持1-200字符 |
| 语言 | 下拉选择 | ✓ | 生成内容的语言（中文/English） |
| 演讲时长 | 滑块/选择 | ✓ | 5/10/15/20/30分钟可选 |

**演讲时长与页数对应关系**：
| 时长 | 预计页数 | 每页演讲时间 |
|------|----------|--------------|
| 5分钟 | 5-6页 | ~1分钟/页 |
| 10分钟 | 8-10页 | ~1分钟/页 |
| 15分钟 | 12-15页 | ~1分钟/页 |
| 20分钟 | 16-20页 | ~1分钟/页 |
| 30分钟 | 24-30页 | ~1分钟/页 |

#### 2.2.2 AI处理模块

**阶段一：资料收集**
- 基于用户输入的主题进行联网搜索
- 收集相关的事实、数据、观点
- 整理筛选高质量信息源

**阶段二：大纲规划**
- 根据演讲时长确定PPT结构
- 生成逻辑清晰的内容大纲
- 包含：封面、目录、正文各节、总结、致谢

**阶段三：内容生成**
- 为每页生成标题和要点
- 填充详细内容和数据支撑
- 确保内容连贯、逻辑通顺

**阶段四：PPT生成**
- 应用简洁专业的设计模板
- 自动排版布局
- 生成PPTX文件

#### 2.2.3 输出模块

- **预览功能**：生成完成后展示大纲预览
- **下载功能**：一键下载PPTX文件
- **历史记录**：在localStorage中保存生成历史

### 2.3 非功能需求

#### 2.3.1 性能需求
- 生成过程提供实时进度反馈
- 单次生成建议在2分钟内完成

#### 2.3.2 兼容性需求
- 支持主流现代浏览器（Chrome、Firefox、Safari、Edge）
- 响应式设计，支持桌面端和移动端访问

#### 2.3.3 存储需求
- 使用浏览器localStorage存储历史记录
- 单条记录存储：主题、时间、大纲、下载链接

## 3. 页面结构

### 3.1 页面清单

| 页面 | 路由 | 说明 |
|------|------|------|
| 首页 | `/` | 输入表单页面 |
| 生成页 | `/generate` | 显示生成进度 |
| 结果页 | `/result` | 预览和下载 |
| 历史页 | `/history` | 查看历史记录 |

### 3.2 页面流转

```
首页(输入) → 生成页(处理中) → 结果页(预览/下载)
                                    ↓
                               历史页(可选)
```

## 4. 技术架构

### 4.1 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Next.js 14+ (App Router) |
| UI组件 | Tailwind CSS + Shadcn/ui |
| 状态管理 | React Context / Zustand |
| 数据存储 | localStorage |
| AI服务 | OpenAI API / Claude API |
| 联网搜索 | Tavily API / SerpAPI |
| PPT生成 | PptxGenJS |

### 4.2 API设计

#### POST `/api/generate`

**请求体**：
```json
{
  "topic": "人工智能在医疗领域的应用",
  "language": "zh-CN",
  "duration": 15
}
```

**响应体**（流式）：
```json
{
  "stage": "collecting | outlining | generating | completed",
  "progress": 0-100,
  "message": "正在收集资料...",
  "data": {
    "outline": [...],
    "downloadUrl": "..."
  }
}
```

## 5. 数据结构

### 5.1 生成记录

```typescript
interface GenerationRecord {
  id: string;                    // 唯一ID
  topic: string;                 // 主题
  language: 'zh-CN' | 'en-US';   // 语言
  duration: number;              // 时长（分钟）
  outline: OutlineItem[];        // 大纲
  createdAt: string;             // 创建时间
  downloadUrl?: string;          // 下载链接
}

interface OutlineItem {
  title: string;                 // 页标题
  points: string[];              // 要点列表
}
```

## 6. 用户体验设计原则

### 6.1 设计原则
- **简洁至上**：界面清爽，无冗余元素
- **进度透明**：实时展示AI工作状态
- **快速上手**：无需登录，即开即用
- **反馈及时**：操作即响应，状态清晰

### 6.2 视觉风格
- 现代简约风格
- 主色调：深蓝色系 (#1a365d)
- 辅助色：浅灰背景 + 白色卡片
- 字体：系统默认无衬线字体
